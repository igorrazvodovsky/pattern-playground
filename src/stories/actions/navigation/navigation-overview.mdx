# Navigation models

The selection of a connective topology that shapes how actors move between destinations, balancing freedom of movement against orientation cost. Each model determines which [navigation behaviours](../?path=/docs/foundations-intent-interaction--docs#navigation) are possible and how [agency](../?path=/docs/qualities-agency--docs) is distributed.

## Models

- [Hub and spoke](../?path=/docs/actions-navigation-hub-and-spoke--docs) — central hub lists all sections, actors return to hub for cross-section navigation
- [Overview and detail](../?path=/docs/actions-navigation-overview-and-detail--docs) — simultaneous display of collection and item (Split View)
- [Fully connected](../?path=/docs/actions-navigation-fully-connected--docs) — every page links to all others through global navigation
- [Fully connected](../?path=/docs/actions-navigation-fully-connected--docs) — every page links to all others through global navigation
- [Multilevel tree](../?path=/docs/actions-navigation-multilevel-tree--docs) — hierarchical structure with global navigation for main pages, local navigation for subpages
- [Step by step](../?path=/docs/actions-navigation-step-by-step--docs) — prescribed sequences with back/next controls
- [Pyramid](../?path=/docs/actions-navigation-pyramid--docs) — hub lists sequence, actors can navigate sequentially or jump to any item
- [Pan and zoom](../?path=/docs/actions-navigation-pan-and-zoom--docs) — continuous spatial navigation through large single spaces
- [Flat navigation](../?path=/docs/actions-navigation-flat-navigation--docs) — minimal navigation, all functions accessible in one place
- [Hybrid patterns](../?path=/docs/actions-navigation-hybrid-patterns--docs) — combinations of multiple models

### Refining by behaviour

- Navigate to known locations → [Fully connected](../?path=/docs/actions-navigation-fully-connected--docs), [tree](../?path=/docs/actions-navigation-multilevel-tree--docs)
- Browse and discover → [Pyramid](../?path=/docs/actions-navigation-pyramid--docs), [pan and zoom](../?path=/docs/actions-navigation-pan-and-zoom--docs)
- Complete focused tasks → [Step by step](../?path=/docs/actions-navigation-step-by-step--docs), [flat](../?path=/docs/actions-navigation-flat-navigation--docs)
- Monitor multiple areas → [Hub and spoke](../?path=/docs/actions-navigation-hub-and-spoke--docs)

### Refining by agency

- Maximum control → [Fully connected](../?path=/docs/actions-navigation-fully-connected--docs), [pan and zoom](../?path=/docs/actions-navigation-pan-and-zoom--docs), [flat](../?path=/docs/actions-navigation-flat-navigation--docs)
- Balanced guidance → [Pyramid](../?path=/docs/actions-navigation-pyramid--docs), [tree](../?path=/docs/actions-navigation-multilevel-tree--docs)
- System-guided → [Hub and spoke](../?path=/docs/actions-navigation-hub-and-spoke--docs), [step by step](../?path=/docs/actions-navigation-step-by-step--docs)

### Refining by scale

- Small (< 10) → [Fully connected](../?path=/docs/actions-navigation-fully-connected--docs)
- Medium (10-50) → [Hub](../?path=/docs/actions-navigation-hub-and-spoke--docs) (mobile), [tree](../?path=/docs/actions-navigation-multilevel-tree--docs) (desktop), [pyramid](../?path=/docs/actions-navigation-pyramid--docs) (sequences)
- Large (50+) → [Hybrid patterns](../?path=/docs/actions-navigation-hybrid-patterns--docs)

### Hybrid combinations

When single models don't fit:

- Hub + fully connected — hub feels constraining, cross-section workflows frequent
- Tree + fully connected — large volume with clear sections, cross-section relationships
- Step by step + pyramid — logical sequence but flexible ordering, varying expertise
- Fully connected + pan and zoom — mix discrete and spatial content
- Hub + flat — multiple independent workspaces, each feature-rich
- Pyramid + tree — many sequences requiring category organisation

## Malleable Routing

In a [malleable system](../?path=/docs/qualities-malleability--docs), the target of a link is not a fixed location. Users can choose *how* they want to open a resource, affecting the [composition](../?path=/docs/qualities-malleability--docs#3-composition-topology) of their workspace:

*   **Standard**: Navigate to new page (Replace current context).
*   **Split**: Open side-by-side (Compare contexts).
*   **Peek**: Open in overlay/modal (Temporary reference).
*   **Background**: Open in background tab (Queue for later).

Navigation becomes a tool for composition.

## Implementation considerations

### Universal navigation features

All models benefit from:
- [Deep linking](../?path=/docs/operations-deep-linking--docs) for addressability
- [Searching](../?path=/docs/actions-seeking-searching--docs) for direct access
- [Breadcrumbs](../?path=/docs/operations-breadcrumbs--docs) for context
- [Command menu](../?path=/docs/actions-seeking-command-menu--docs) for power users

### UI design independence

Models can be rendered in various ways:
- Tabs, menus, sidebar trees for *fully connected* or *tree*
- Cards, lists, grids for *hub and spoke*
- Progress indicators, stepper components, or next/previous for *step by step*
