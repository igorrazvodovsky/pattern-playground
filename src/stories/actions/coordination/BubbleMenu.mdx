# Bubble menu

The surfacing of relevant actions at the point of selection, balancing immediate capability with uninterrupted flow. Currently realised within the [block-based editor](../?path=/docs/actions-sensemaking-blockbased-editor--docs); the pattern extends to any context where content is selectable.

## Related patterns

### Precursors

- [Selection](../?path=/docs/actions-coordination-selection--docs) — the bubble menu exists because content has been selected; the content-range variant of selection is what it consumes
- [Progressive disclosure](../?path=/docs/actions-seeking-progressive-disclosure--docs)

### Complementary

- [Popover](../?path=/docs/operations-popover--docs) - underlying positioning mechanism
- [Dropdown](../?path=/docs/actions-coordination-dropdown--docs) - provides the container mechanism; bubble menu adds selection-awareness and contextual action composition
- [Action bar](../?path=/docs/actions-coordination-action-bar--docs) — the globally-positioned sibling; both are summoned by selection, but the action bar scopes to a set of items in a list or grid rather than to content inside an editor
- [Command menu](../?path=/docs/actions-seeking-command-menu--docs) - alternative access to actions
- [Context menu](../?path=/docs/actions-application-context-menu--docs) - pointer-invoked alternative; both are transient action surfaces
