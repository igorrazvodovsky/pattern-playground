# Dialog

The interruption of a workflow to demand a focused decision, balancing urgency against disruption.

<Story of={ DialogStories.Default } />

## Scrolling
By design, a dialog’s height will never exceed that of the viewport. As such, dialogs will not scroll with the page ensuring the header and footer are always accessible to the user.

<Story of={ DialogStories.Scrolling } />


## Related patterns

Dialog is a [transient mode](../?path=/docs/foundations-intent-interaction--docs#modes) — it suspends the primary workflow to force a focused decision. There is also significant overlap between it, [popover](../?path=/docs/operations-popover--docs) and [drawer](../?path=/docs/actions-application-drawer--docs).

A dialog can be used for presenting critical [notifications](./?path=/docs/actions-coordination-notification--docs) that require user action.
