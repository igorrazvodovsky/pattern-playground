# Next-best action

The system analyses what the actor is doing and recommends the most useful next step.

The most forward-leaning pattern on the [assisted task completion](../?path=/docs/actions-application-assisted-task-completion--docs) spectrum, right at the edge of autonomous action. It works at the *workflow level*, not at the field or content level, and suggests *actions* — schedule a call, run tests, send a follow-up — rather than data values or text.

{/* ## When it works */}

{/* - The workflow has clear sequential steps and the system has strong contextual signals: CRM after logging a call, IDE after editing code, onboarding flows */}
{/* - Recommendations appear at natural *transition points* between workflow steps, not mid-task */}
{/* - The suggestion includes a brief rationale. Actors need to understand *why* this action is recommended, or they won't act on it */}

{/* ## When it fails */}

{/* - Recommendations feel presumptuous or interruptive. This is the "annoying assistant" trap, and it's easy to fall into */}
{/* - The system doesn't have enough context to suggest anything useful */}
{/* - Suggestions pile up without resolution, and actors start ignoring all of them */}

- [Delegation](../?path=/docs/foundations-delegation--docs) — the action can also be suggested to the user by the agent
- [Agency](../?path=/docs/qualities-agency--docs) — proactive recommendations must preserve actor control
- [Adaptability](../?path=/docs/qualities-adaptability--docs) — recommendations that improve from accepted/rejected signal over time

### Precursors

- [Suggestion](../?path=/docs/actions-application-suggestion--docs) — the broader pattern; next-best action is specifically about recommending workflow steps
- [Wizard](../?path=/docs/actions-application-wizard--docs) — prescribed sequences; next-best action is the adaptive, context-aware alternative
