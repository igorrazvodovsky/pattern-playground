## Related patterns

### Complementary
- [Autocomplete](../?path=/docs/operations-autocomplete--docs) — narrows the option space as the actor types, when a known set exists
- [Autofill](../?path=/docs/operations-autofill--docs) — supplies the value the input would otherwise have to ask for
- [Good defaults](../?path=/docs/operations-good-defaults--docs) — pre-populates the field so the actor only intervenes when the guess is wrong
- [Suggestion](../?path=/docs/actions-application-suggestion--docs) — offers candidate values the input can adopt, modify, or reject

### Follow-ups
- [Form](../?path=/docs/actions-application-form--docs) — the container that groups inputs into a coherent ask
- [Data entry](../?path=/docs/actions-application-data-entry--docs) — the broader activity input contributes to, alongside format and friction choices
- [Searching](../?path=/docs/actions-seeking-searching--docs) — where the input becomes the entry point for query formulation

### Tangentially related
- [Textarea](../?path=/docs/operations-textarea--docs) — chosen instead when the answer doesn't fit a single line
- [Range](../?path=/docs/operations-range--docs) — chosen instead when the value is bounded and better committed by direct manipulation than by typing
