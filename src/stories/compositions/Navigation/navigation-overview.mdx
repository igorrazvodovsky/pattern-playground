# Navigation models

Structural patterns describing how screens, pages, or spaces connect to one another. These models shape which [navigation behaviours](../?path=/docs/foundations-intent-interaction--docs#navigation) actors can perform and how [agency](../?path=/docs/qualities-agency--docs) is distributed.

## Models

- [Hub and spoke](../?path=/docs/compositions-structure-hub-and-spoke--docs) ‚Äî central hub lists all sections, actors return to hub for cross-section navigation
- [Overview and detail](../?path=/docs/compositions-structure-overview-and-detail--docs) ‚Äî simultaneous display of collection and item (Split View)
- [Fully connected](../?path=/docs/compositions-structure-fully-connected--docs) ‚Äî every page links to all others through global navigation
- [Fully connected](../?path=/docs/compositions-structure-fully-connected--docs) ‚Äî every page links to all others through global navigation
- [Multilevel tree](../?path=/docs/compositions-structure-multilevel-tree--docs) ‚Äî hierarchical structure with global navigation for main pages, local navigation for subpages
- [Step by step](../?path=/docs/compositions-structure-step-by-step--docs) ‚Äî prescribed sequences with back/next controls
- [Pyramid](../?path=/docs/compositions-structure-pyramid--docs) ‚Äî hub lists sequence, actors can navigate sequentially or jump to any item
- [Pan and zoom](../?path=/docs/compositions-structure-pan-and-zoom--docs) ‚Äî continuous spatial navigation through large single spaces
- [Flat navigation](../?path=/docs/compositions-structure-flat-navigation--docs) ‚Äî minimal navigation, all functions accessible in one place
- [Hybrid patterns](../?path=/docs/compositions-structure-hybrid-patterns--docs) ‚Äî combinations of multiple models

### Refining by behaviour

- Navigate to known locations ‚Üí [Fully connected](../?path=/docs/compositions-structure-fully-connected--docs), [tree](../?path=/docs/compositions-structure-multilevel-tree--docs)
- Browse and discover ‚Üí [Pyramid](../?path=/docs/compositions-structure-pyramid--docs), [pan and zoom](../?path=/docs/compositions-structure-pan-and-zoom--docs)
- Complete focused tasks ‚Üí [Step by step](../?path=/docs/compositions-structure-step-by-step--docs), [flat](../?path=/docs/compositions-structure-flat-navigation--docs)
- Monitor multiple areas ‚Üí [Hub and spoke](../?path=/docs/compositions-structure-hub-and-spoke--docs)

### Refining by agency

- Maximum control ‚Üí [Fully connected](../?path=/docs/compositions-structure-fully-connected--docs), [pan and zoom](../?path=/docs/compositions-structure-pan-and-zoom--docs), [flat](../?path=/docs/compositions-structure-flat-navigation--docs)
- Balanced guidance ‚Üí [Pyramid](../?path=/docs/compositions-structure-pyramid--docs), [tree](../?path=/docs/compositions-structure-multilevel-tree--docs)
- System-guided ‚Üí [Hub and spoke](../?path=/docs/compositions-structure-hub-and-spoke--docs), [step by step](../?path=/docs/compositions-structure-step-by-step--docs)

### Refining by scale

- Small (< 10) ‚Üí [Fully connected](../?path=/docs/compositions-structure-fully-connected--docs)
- Medium (10-50) ‚Üí [Hub](../?path=/docs/compositions-structure-hub-and-spoke--docs) (mobile), [tree](../?path=/docs/compositions-structure-multilevel-tree--docs) (desktop), [pyramid](../?path=/docs/compositions-structure-pyramid--docs) (sequences)
- Large (50+) ‚Üí [Hybrid patterns](../?path=/docs/compositions-structure-hybrid-patterns--docs)

### Hybrid combinations

When single models don't fit:

- Hub + fully connected ‚Äî hub feels constraining, cross-section workflows frequent
- Tree + fully connected ‚Äî large volume with clear sections, cross-section relationships
- Step by step + pyramid ‚Äî logical sequence but flexible ordering, varying expertise
- Fully connected + pan and zoom ‚Äî mix discrete and spatial content
- Hub + flat ‚Äî multiple independent workspaces, each feature-rich
- Pyramid + tree ‚Äî many sequences requiring category organisation

## Malleable Routing

In a [malleable system](../?path=/docs/qualities-malleability--docs), the target of a link is not a fixed location. Users can choose *how* they want to open a resource, affecting the [composition](../?path=/docs/qualities-malleability--docs#3-composition-topology) of their workspace:

*   **Standard**: Navigate to new page (Replace current context).
*   **Split**: Open side-by-side (Compare contexts).
*   **Peek**: Open in overlay/modal (Temporary reference).
*   **Background**: Open in background tab (Queue for later).

Navigation becomes a tool for [composition](../?path=/docs/compositions-composition-canvas--docs).

## Implementation considerations

### Universal navigation features

All models benefit from:
- [Deep linking](../?path=/docs/primitives-deep-linking--docs) for addressability
- [Searching](../?path=/docs/compositions-searching--docs) for direct access
- [Breadcrumbs](../?path=/docs/patterns-breadcrumbs--docs) for context
- [Command menu](../?path=/docs/patterns-command-menu--docs) for power users

### UI design independence

Models can be rendered in various ways:
- Tabs, menus, sidebar trees for *fully connected* or *tree*
- Cards, lists, grids for *hub and spoke*
- Progress indicators, stepper components, or next/previous for *step by step*
