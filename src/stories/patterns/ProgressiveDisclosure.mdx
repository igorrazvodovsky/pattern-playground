# Progressive disclosure

Prioritize key information and core features while deferring secondary elements to additional views.
The approach prevents overwhelming users when presenting large amounts of content.
By revealing hidden elements only when triggered, you maintain full functionality while significantly reducing
initial cognitive load and creating a cleaner interface.

{/* ### Scaffolding via *greying out* */}
{/* When designing layered interfaces, prefer **greying out** (ghosting) unavailable features over hiding them completely. */}
{/* - **Why:** Seeing the disabled controls helps users build a mental model of *where* advanced features live. */}
{/* - **Result:** Users know the capability exists, reducing the need to re-learn the interface when they transition to the expert layer. */}

{/* # Best practices

- Clearly distinguishing between the primary and secondary tasks.
- Creating clear trigger actions for secondary content.
- Minimising nested disclosures; simplifing features with multiple levels.*/}
