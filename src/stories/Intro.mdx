<h2>Objective and approach</h2>

Using the term "design system" is both helpful and misleading. Building a complete design system requires the context of a product or organisation, which this project lacks.
This work resembles ongoing design systems practice, focusing on process and workflow improvements through decomposition. *The project uses design practice itself as a target for design-system-style organisation*.
{/* Is it weird? */}

In practical terms the objective is to identify and describe smart, reusable patterns while approaching the process like tending a garden &mdash; cultivating ideas over time, refining structures and fostering connections.
{/* TODO: 'garden' needs clarification */}

{/* From a design process perspective, I'm trying to keep an eye on topic of maintenance... */}

On a technical level, it is also a learning exercise. Not all patterns follow the same implementation strategy. Some are kept as CSS-only, others are made into web and React components.
The aim is to look for structures that are easy to use both in design tools and code. This approach can simultaneously help designers and developers collaborate, help with pattern adoption, and make them more accessible for contribution.

## On LLMs and AI

I didn't set out to make this an AI project. The plan was to stay technology-neutral. But in reality of 2025 it's become a playground for weaving LLM patterns into component-based structures, using design systems as a lens to explore what's emerging.
